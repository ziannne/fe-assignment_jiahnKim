# 쇼핑몰 앱 프로젝트

## 프로젝트 소개
이 프로젝트는 React 기반의 쇼핑몰 웹앱으로, 상품을 둘러보고 상세 정보를 확인할 수 있도록 구성한 SPA입니다. 홈 화면에서는 상품 목록을 보고, 상품을 선택하면 해당 상품의 상세 페이지로 이동합니다. 또, 상품에 대한 리뷰도 함께 확인할 수 있습니다.

이 흐름은 단순한 화면 구성만으로 끝나는 것이 아니라, 서버에서 데이터를 가져오고 페이지를 이동하는 구조까지 함께 구현한 프로젝트입니다.

---

## 핵심 흐름
사용자가 홈 화면에 접속하면 상품 목록을 불러옵니다. 상품을 클릭하면 상세 페이지로 이동하고, 해당 상품의 정보와 리뷰를 다시 요청합니다. 요청이 끝나면 화면에 데이터를 보여주고, 데이터가 없거나 실패한 경우에는 적절한 안내를 표시합니다.

이 과정을 한 문장으로 정리하면, 사용자가 보고 싶은 정보를 요청하면 앱이 그에 맞는 데이터를 가져와 화면에 보여주는 구조입니다.

---

## 비동기 통신과 API
브라우저에서 서버에 데이터를 요청하면 바로 결과를 받을 수 없기 때문에 비동기 처리가 필요합니다. 이 프로젝트에서는 `fetch`와 `async/await`를 사용해 데이터를 가져왔습니다.

데이터는 로컬 서버인 `json-server`를 통해 제공했습니다. 상품 정보와 리뷰 정보는 `server/db.json`에 저장되어 있고, 프론트엔드는 이 데이터를 API 형태로 요청합니다.

이 과정은 다음과 같이 이어집니다.

- 홈 화면에서 상품 목록 요청
- 상세 페이지에서 특정 상품 정보 요청
- 해당 상품의 리뷰 요청

즉, 화면은 사용자에게 보여주는 역할을 하고, 데이터는 API를 통해 받아오는 구조로 구성했습니다.

---

## 라우팅
사용자가 상품을 클릭하면 상세 페이지로 이동해야 합니다. 이때 페이지를 새로고침하지 않고 URL만 바꾸며 화면을 전환하는 방식이 라우팅입니다. 이 프로젝트에서는 `react-router-dom`을 사용해 URL에 따라 다른 화면을 보여주도록 구성했습니다.

홈에서 상품을 누르면 상세 페이지로 이동하고, 주소도 그에 맞게 바뀝니다. 상세 페이지에서는 URL에 포함된 상품 번호를 기준으로 해당 상품 데이터를 요청합니다. 이 구조 덕분에 페이지 이동과 데이터 요청이 자연스럽게 연결됩니다.

## 프로젝트 구성
### 화면 구성
- 홈 화면
- 상품 상세 화면
- 공통 레이아웃
- 로딩 및 에러 상태 표시

### 데이터 처리
- 상품 목록 조회
- 상품 상세 조회
- 리뷰 조회
- 로딩/에러 상태 관리

### 사용자 경험
- 클릭하면 자연스럽게 상세 페이지로 이동
- 데이터 요청 중에는 로딩 상태 표시
- 서버 응답이 없거나 실패하면 안내 메시지 출력

## 프로젝트 구조
```text
src/
├─ components/
│  ├─ Layout/
│  ├─ Home/
│  ├─ About.jsx
│  ├─ NotFound.jsx
│  └─ ProductDetail.jsx
├─ hooks/
│  ├─ useFetchProducts.js
│  ├─ useFetchProduct.js
│  └─ useFetchReview.js
├─ App.jsx
├─ main.jsx
└─ index.css
```

- `components` : 화면 UI를 구성하는 컴포넌트
- `hooks` : API 요청과 데이터 처리를 담당하는 커스텀 훅
- `App.jsx` : 라우팅 설정

## 실행 방법
1. 의존성 설치
   ```bash
   npm install
   ```
2. 개발 서버 실행
   ```bash
   npm run start:dev
   ```
3. 브라우저에서 접속
   - 프론트엔드: http://localhost:5177
   - 로컬 API: http://localhost:3000

이 프로젝트는 단순한 쇼핑몰 UI를 넘어, 실제 웹앱에서 자주 쓰이는 데이터 요청과 페이지 이동 구조를 직접 구현한 프로젝트입니다.
